- POJO (Plain Old Java Object):

+ Là những Java object không bị hạn chế bởi bất cứ thứ gì khác (nghĩa là không hiện thực hay được mở rộng từ bất kỳ một class nào khác và cũng không chứa bất kỳ annotation nào.

+ Chỉ chứa các thuộc tính, các phương thức setter, getter của những thuộc tính đó (có thể có thêm cả các phương thức override toString() và equals()).

- JavaBean:

+ Là POJO.

+ Có ít nhất một constructor không có tham số.

+ Implement interface Serializable.

- ORM: (viết tắt của Object Relational Mapping) là một công nghệ/ khái niệm/ quá trình chuyển đổi dữ liệu từ ngôn ngữ hướng đối tượng sang cơ sở dữ liệu quan hệ và ngược lại. Ví dụ, trong Java nó được thực hiện với sự trợ giúp của Reflection và JDBC.

- JPA: (viết tắt của Java Persistence API) là một chuẩn đặc tả cho việc ánh xạ giữa các đối tượng Java và cơ sở dữ liệu quan hệ thông qua công nghệ ORM (Object Relational Mapping).

Entity: là các đối tượng thể hiện tương ứng một table trong cơ sở dữ liệu.
