Giao diện (Interfaces)

  1. Trang chủ
  2. Java
  3. Giao diện (Interfaces)

Giao diện là một lớp hoàn toàn trừu tượng chỉ chứa các phương thức trừu tượng.

Một số thông số kỹ thuật cho giao diện:

- Xác định bằng cách sử dụng từ khóa interface.

- Có thể chỉ chứa các biến static, public, final.

- Không thể chứa một constructor.

- Giao diện có thể mở rộng các giao diện khác.

Ví dụ:

interface Animal {{ '{' }}
  public void eat();
  public void makeSound();
{{ '}' }}

Giao diện có các thuộc tính sau:

- Một giao diện được ngầm hiểu là trừu tượng. Bạn không cần phải sử dụng từ khoá abstract trong khi khai báo một giao diện.

- Mỗi phương thức trong một giao diện cũng được ngầm hiểu là trừu tượng, vì vậy từ khoá abstract là không cần thiết.

- Phương thức trong một giao diện được ngầm hiểu là public.

Một lớp có thể kế thừa từ một lớp cha, nhưng có thể thực hiện nhiều giao diện!

Sử dụng từ khóa implements để kế thừa từ các interface.

interface Animal {{ '{' }}
    public void eat();
    public void makeSound();
{{ '}' }}

class Cat implements Animal {{ '{' }}
    public void makeSound() {{ '{' }}
        System.out.println("Meow");
    {{ '}' }}
    public void eat() {{ '{' }}
        System.out.println("omnomnom");
    {{ '}' }}
{{ '}' }}

public class Program {{ '{' }}
    public static void main(String[] args) {{ '{' }}
        Cat c = new Cat();
        c.eat();
    {{ '}' }}
{{ '}' }}

/***********
 * Output
 * omnomnom
*************/

Khi bạn thực hiện một giao diện, bạn cần phải ghi đè tất cả các phương thức của nó.
