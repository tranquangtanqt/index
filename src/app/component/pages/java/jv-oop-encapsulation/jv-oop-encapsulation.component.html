Ý tưởng đằng sau đóng gói là để đảm bảo rằng những chi tiết thực hiện không hiển thị cho người dùng. Các biến của một lớp sẽ bị ẩn khỏi các lớp khác, chỉ có thể truy cập thông qua các phương thức của lớp hiện tại. Đây được gọi là dữ liệu ẩn.

Để đạt được sự đóng gói trong Java, khai báo các biến của lớp là private và cung cấp phương thức setter và getter chung để sửa đổi và xem các giá trị của các biến.

public class Student {{ '{' }}
    private String name;
 
    public String getName() {{ '{' }}
        return name;
    {{ '}' }}
 
    public void setName(String name) {{ '{' }}
        this.name = name;
    {{ '}' }}
{{ '}' }}

class Test {{ '{' }}
    public static void main(String[] args) {{ '{' }}
        Student s = new Student();
        s.setName("Nguyen Van A");
        System.out.println(s.getName());
    {{ '}' }}
{{ '}' }}

Tóm lại, đóng gói cung cấp các lợi ích sau:

- Kiểm soát truy cập hoặc sửa đổi dữ liệu

- Mã linh hoạt và dễ thay đổi hơn

- Khả năng thay đổi một phần của mã mà không ảnh hưởng đến các phần khác
