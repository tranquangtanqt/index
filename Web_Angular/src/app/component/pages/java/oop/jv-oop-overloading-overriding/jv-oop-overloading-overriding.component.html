Nạp chồng phương thức và ghi đè phương thức (overloading & overriding)

  1. Trang chủ
  2. Java
  3. Nạp chồng phương thức và ghi đè phương thức (overloading & overriding)

1. Ghi đè phương thức (overriding): xảy ra nếu lớp con có phương thức giống lớp cha.

- Ghi đè phương thức cũng được gọi là đa hình lúc runtime.

- Nguyên tắc ghi đè phương thức:

1. Phương thức phải có tên giống với lớp cha.

2. Phương thức phải có tham số giống với lớp cha.

3. Lớp con và lớp cha có mối quan hệ kế thừa.

class Animal {{ '{' }}
    void eat() {{ '{' }}
        System.out.println("animal...");
    {{ '}' }}
{{ '}' }}
 
class Dog extends Animal {{ '{' }}
    void eat() {{ '{' }}
        System.out.println("dog...");
    {{ '}' }}
{{ '}' }}
 
class Program {{ '{' }}
    public static void main(String args[]) {{ '{' }}
        Animal a = new Dog();
        a.eat();
    {{ '}' }}
{{ '}' }}

/*********
 * Output
 * dog...
*********/

Lưu ý:

- Không thể ghi đè bằng phương thức final hoặc static.

- Không thể ghi đè Constructors.

- Ghi đè phương thức thì tham số và kiểu trả về phải giống nhau.

- Chỉ ghi đè phương thức, không ghi đè biến.

Ví dụ:

class Bike{{ '{' }}  
    public int speedlimit = 90;  
{{ '}' }}  
class Honda3 extends Bike{{ '{' }}  
    public int speedlimit = 150;  
   
 public static void main(String args[]){{ '{' }}  
    Bike obj = new Honda3();  
    System.out.println(obj.speedlimit);
    {{ '}' }} 
{{ '}' }}

/***********
 * Output
 * 90
***********/

Xét ví dụ:

class Animal {{ '{' }}
    void eat() {{ '{' }}
        System.out.println("animal...");
    {{ '}' }}
{{ '}' }}
 
class Dog extends Animal {{ '{' }}
    void eat() {{ '{' }}
        System.out.println("dog...");
    {{ '}' }}
{{ '}' }}
 
class BabyDog extends Dog {{ '{' }}
    
{{ '}' }}

class Program {{ '{' }}
    public static void main(String args[]) {{ '{' }}
        Animal a = new BabyDog();
        a.eat();
    {{ '}' }}
{{ '}' }}

/*********
 * Output
 * dog...
*********/

Vì BabyDog không ghi đè phương thức eat(), nên phương thức eat() của lớp Dog được gọi.

2. Nạp chồng phương thức (overloading): xảy ra nếu một lớp có nhiều phương thức có tên giống nhau nhưng khác nhau về kiểu dữ liệu hoặc số lượng các tham số.

- Nạp chồng phương thức hay còn gọi là đa hình tại complie-time.

- Có 2 cách nạp chồng phương thức:

+ Thay đổi số lượng các tham số

+ Thay đổi kiểu dữ liệu của các tham số

Ví dụ nạp chồng phương thức:

class Program {{ '{' }}
    static double max(double a, double b) {{ '{' }}
        return a > b ? a : b;
    {{ '}' }}
    static int max(int a, int b) {{ '{' }}
        return a > b ? a : b;
    {{ '}' }}

    public static void main(String[] args) {{ '{' }}        
        System.out.println(max(8, 17));
        System.out.println(max(3.14, 7.68));
    {{ '}' }}
{{ '}' }}

/**********
 * Output
 * 17
 * 7.68
**********/
